Overview

I wanted this website to both be an overview of my work, as well as a place to publish more in-depth descriptions of all of the ways I create. It is also a first impression, communicating both my personal style and projects. This design has lived for a while in my notebooks, post-its, and unpublished HTML files stored locally on my computer. This website has changed iteratively over time, but I have created something I am content with, and feel that it reflects my body of work.

Drafts

The first drafts of this website where on paper, using what highlighters I had available on hand as a placeholder to workout the layout without getting caught up in the details. I had played around with stripes of color as dividers across sections (like horizontal rulings). Ultimately, I liked the retro, mod feel of the connected stripes weaving through the page in this sketch: [IMG HERE] I love color. However, bright colors can easily be overwhelming. I wanted this website to have pops of color, while still being easily readable. I borrow inspiriation from the color palettes of retro mod-style print from the 60s and 70s, pairing bright colors with warm neutrals to create a pleasing analogous color scheme.

Prototypes

[THUMBNAIL] Within this prototype I was able to tweak the colors for nice contrast against each other as well as the screen. At this point, I did not know how exactly I was going to accomplish the border stripes, intially assuming I would have to create custom SVG images and align them as a border image on the div elements. I sought out some blogs that cover niche and interesting ways that CSS borders, box-shadows, and outlines ([such as this blog] or [this blog]) can be used to create multi-layered stripes. Thankfully, I was able to create the effect I wanted using pure CSS, which I always prefer for better cross-compatability and to reduce the number of objects that might fail or break. I am pleased that I was able to use pure HTMl and CSS for all of the website features and aesthetics. For instance, if you use your cursor to highlight my name at the top of the navbar, it is embedded text that can be copy and pasted (and read by screen readers).

Final

If you're reading this, you are on the final and live version of this website. :) For this website, I was aiming for elegance in both the aesthetics and the functionality. Aesthetically, I chose bright colors and contrasted on a dark background for something that reads like a dark-mode theme (on a browser or code editor). Functionally, I crafted the website to be easy to read and logical to navigate. I was careful of the information system, with pages that start as big overviews, and link down to specific projects with more detail for those who seek it. I wanted high-contrast against the background and text, and I chose to use the font Atkinson Hyperlegible for both ease of use and accessibility. I kept the back-end simple and use plain HTML/CSS wherever possible (such as the pure-CSS borders and embedded fonts). I find that each project needs different frameworks and environments depending on the functionality, and I try to keep complexity to a minimum wherever possible.
